## 一、当前架构存在的问题

### 1. 代码层面：历史包袱太重
| 问题 | 举例 | 影响 |
|------|------|------|
| **Windows 遗产** | scripts/ 中的 .bat/.ps1、路径硬编码 | 不可执行，纯占空间 |
| **冗余版本** | BTDR 有 v1/v2/v3/v21/v22/... 多个版本文件夹 | 分不清哪个是实盘哪个是归档 |
| **旧工具未清理** | token_monitor.py、find_token_y.py、capture_full.py 等qclaw专属脚本 | 永远不会在OpenClaw上运行 |
| **策略散落** | `strategies/量化AI公司/` 嵌套深、文件名无统一规范 | 难以维护和扩展 |

### 2. 架构层面：OpenClaw 能力未充分利用
| OpenClaw 原生能力 | 当前用法 | 应该怎么做 |
|------------------|---------|-----------|
| **TaskFlow** (持久化任务) | 未使用 | 策略执行用 TaskFlow 队列 |
| **memory-core dreaming** | 仅开配置 | 应主动喂策略决策日志 |
| **sessions_spawn** (子智能体) | 未使用 | 三路投票抗幻觉用子智能体 |
| **cron** | 4个基础任务 | 可增加收盘后回测触发 |
| **message 工具** | 未配置WeCom通道 | 设置 delivery.channel |
| **web_search** | 可用 | 接入市场情报自动化 |

### 3. 流程层面：缺少自动化闭环
```
当前: 人工指令 → 执行 → 人工查看结果
目标: 市场事件 → 自动分析 → 推送建议 → 人工确认/自动执行
```

## 二、关键优化方向（按优先级）

### P0: 代码瘦身（1-2天）
```
cleanup/
├── 删除不可执行的Windows脚本 (.bat/.ps1)
├── 归档旧版本策略（保留最新2版，其余移入archive/）
├── 清理qclaw专用工具（token监控/OCR/QClaw UI相关）
├── 统一策略入口：scripts/strategy_runner.py
└── 标准化日志输出到 data/logs/
```

### P1: OpenClaw 深度集成（1周）
```
integrate/
├── 策略执行改用 TaskFlow（失败自动重试+状态持久化）
├── 子智能体三路投票（sessions_spawn 技术面/基本面/风控）
├── 记忆系统对接（交易决策→memory-core→自动提取经验）
├── 每日收敛：memory-core dreaming → 更新 MEMORY.md
└── 设置 cron delivery.channel 直达企业微信
```

### P2: 自动化交易闭环（2周）
```
automate/
├── 盘前：market_briefing.py → 自动推送摘要
├── 盘中：心跳巡检 → 风控阈值检查 → 异常推送
├── 收盘：读取Today_PNL → 生成日报 → 推送
├── 盘后：自动回测最新数据 → 对比历史表现
└── 进化：对比结果写入 MEMORY.md → dreaming提取
```

### P3: 学习→策略反馈管道（持续）
```
learn/
├── 每周 GitHub 学习 → 提取1-2个可借鉴因子
├── 因子回测验证 → 通过则加入候选因子池
├── 候选因子 → 组合优化 → 策略微调
└── 微调后策略 → 影子模式运行 → 对比实盘
```
