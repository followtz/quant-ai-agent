# 回测驱动的优化方案
**来源**: BTDR 250天回测 + 连连数字65天回测 | **日期**: 2026-05-04

---

## 核心发现

### BTDR 真相
```
波动率 456%/年 = 日均波动 ~2.8%
这意味着: 每10个交易日就有1天涨>5%或跌>5%
          用固定±12%/±5%的阈值，在趋势行情中容易过早离场
```

### PrevClose V2 的问题
| 当前设计 | 实际表现 | 根因 |
|---------|---------|------|
| Turbo A: 卖出+12% | 趋势市中卖飞了 | 456%年波动率，+12%一天就能到，涨了40%时你已经离场了 |
| Turbo A: 买回-1% | 卖出后很难接回 | 在急涨行情中回调不够1% |
| Turbo B: 买入-5% | 信号太多（164次） | 5%的跌幅在日波动2.8%的标的中太常见 |
| Turbo B: 卖出+5% | 买入后小涨就卖 | 错过后续更大涨幅 |

### 双重确认的价值
```
V3信号:    164次 (太频繁，噪音多)
均值回归:   33次 (偏少)
双重确认:  33次 (黄金区间，每7天1次信号)
结论: 双重确认机制是正确的方向
      但阈值需要匹配标的的实际波动率
```

## 优化方案（按优先级）

### P0: 阈值从固定值改为ATR动态值 (今日可做)
```
当前:                 优化后:
Turbo A 卖出 +12% →  ATR × 3.0 (动态)
Turbo A 买回  -1% →  ATR × 0.5 (动态)  
Turbo B 买入  -5% →  ATR × 2.0 (动态)
Turbo B 卖出  +5% →  ATR × 2.0 (动态)

为什么:
  456%年波动 = 日均ATR约$0.32
  ATR×3 = $0.96 ≈ 8-10% (比固定12%更合理)
  ATR会随市场波动自动调整
```

### P1: 回测框架对接实盘 (本周)
```
当前: btdr_monitor.py 读取state.json
优化: auto_backtest.py 直接对接Futu数据
      回测结果自动与实盘对比
```

### P2: 连连数字积累数据 (持续)
```
65天数据不足以做决策
方案: 数据下载cron执行1个月后，再做回测优化
      预计6月初有~90天数据，可以跑有意义的回测
```

### P3: 策略加入Alpha158精选因子 (下月)
```
从精选7个因子中选2-3个，叠加到现有信号上
先用backtrader回测验证有效性
有数据支撑再加入实盘
```
