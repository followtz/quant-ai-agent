## 一、Qlib 核心能力（精读结论）

### Qlib 是什么
> "AI-oriented quantitative investment platform" — 微软出品

### 核心模块
| 模块 | 功能 | 对我们要不要用 |
|------|------|--------------|
| **Data Layer** | 数据获取/清洗/存储 | ✅ 有用的数据治理思路 |
| **Factor Mining** | 158个基础因子 + 自动挖掘 | ✅ 因子库可以借鉴 |
| **Model Training** | LSTM/Transformer/GRU等 | ⚠️ 太重，暂时跳过 |
| **Backtesting** | 回测引擎 | ✅ 可以学习设计理念 |
| **Portfolio** | 组合优化 | ⚠️ 机构级别，暂时略过 |
| **RD-Agent** | LLM自动因子挖掘 | 🔥 有趣但需要大量计算 |

### 对我们的实用性评估（考虑小定位）

```
Qlib适合场景: A股5000+股票、因子挖掘、ML模型
我们场景:     港美股少量标的、规则化交易、轻量化

重叠区域:  因子库灵感(20%) + 数据管线理念(15%) + 其他(5%)
不重叠:     模型训练/组合优化/高频交易(60%)
```

**结论**: Qlib 对我们不是直接工具，而是一个**思想库**。

## 三、Wind 接入方案设计

### 现状
- Wind 安装在你本地 Windows 电脑上
- 无法安装到服务器
- 有高端付费版（数据全面）

### 可行方案：轻量级数据桥

```
你的本地电脑                         云服务器
┌─────────────────┐           ┌──────────────────┐
│  Wind终端        │           │  OpenClaw         │
│    ↓             │  SSH/HTTP │    ↑              │
│  wind_bridge.py  │ ───────→ │  data_receiver.py │
│  (Python脚本)    │   JSON   │  (自动解析存储)     │
│  定时/事件触发    │           │                   │
└─────────────────┘           └──────────────────┘
```

**实现方式**:
1. 在你的电脑上运行一个轻量 Python 脚本
2. 脚本通过 Wind Python API (`from WindPy import w`) 获取数据
3. 数据格式化为 JSON，通过 SSH 隧道或 HTTP POST 发送到服务器
4. 服务器端自动接收并存储到 `data/` 目录

**优势**:
- 不需要在服务器上装 Wind
- 数据格式灵活（只取我们需要的字段）
- 安全性好（你控制数据流出）

## 四、QuantConnect 付费版评估

| 维度 | 评价 | 是否推荐 |
|------|------|---------|
| **云端回测** | 强大的基础设施，但我们的策略规则简单 | ⚠️ 暂时不需要 |
| **数据覆盖** | 全球市场数据，但 Futu OpenD 已有港美股 | ⚠️ 重叠 |
| **策略市场** | 可借鉴他人策略思路 | ✅ 有学习价值 |
| **IB 券商对接** | 支持盈透，但我们用 Futu | ❌ 不兼容 |
| **月费** | 付费版 $xxx/月 | 💰 需评估性价比 |

**初步结论**: 当前阶段不建议订阅。先把手头的策略跑稳，以后如果需要跨市场/跨券商再考虑。
