# AI外脑路由SOP - 典型任务自动路由手册
> 符合SOUL.md第六章AI外脑协同规范，v1.0（2026-04-23）
> 
> **核心原则**：「先风控、再执行、后进化」「外脑建议需经QClaw二次校验」「严禁直接下单」

---

## 一、路由决策树（优先级从高到低）

### 第一层：任务类型分类

```
用户请求/任务触发
       │
       ├── 包含以下关键词？──→ 直接执行/拒绝
       │   「下单」「买入」「卖出」「平仓」「开仓」
       │   「API Key」「密码」「密钥」「账户」
       │   「熔断」「切断」「STOP ALL」「EMERGENCY」
       │   → QClaw自身处理（核心决策/安全敏感/风控类）
       │
       ├── 以下关键词任一触发 ──→ 千问路由
       │   「代码」「Python」「回测」「策略逻辑」「算法」
       │   「review」「审查」「bug」「报错」「修复」
       │   「因子计算」「数学」「统计」「回归」「优化」
       │
       ├── 以下关键词任一触发 ──→ 豆包路由
       │   「新闻」「市场」「行情」「最新」「今日」
       │   「搜索」「情报」「事件」「政策」「公告」
       │   「舆情」「行业」「板块」「竞品」「创意」
       │   「方案」「建议」「改进」「如何」「怎么做」
       │
       ├── 以下关键词任一触发 ──→ 元宝路由
       │   「研报」「报告」「PDF」「文档」「总结」
       │   「摘要」「长文」「解读」「分析」「综合」
       │   「年报」「招股书」「公告全文」「多文档」
       │
       └── 复合任务 ──→ 多助手投票/并行
           「这个策略怎么样」→ 豆包搜索 + 千问代码review + 元宝长文
           「帮我分析这几个标的风险」→ 千问计算 + 元宝研判
           「最近的交易机会」→ 豆包情报 + 千问策略 + 元宝研报
```

### 第二层：安全边界校验（所有外脑任务必须通过）

```
┌──────────────────────────────────────────────┐
│ 安全边界检查（不符合任一则拒绝外脑处理）        │
├──────────────────────────────────────────────┤
│ □ 不含账户信息（持仓金额/成本/盈亏细节）        │
│ □ 不含个人信息（姓名/手机/邮箱/地址）          │
│ □ 不含API密钥/Token/凭证                      │
│ □ 不含交易指令（下单价/数量/买卖方向）          │
│ □ 输出不含直接执行的操作命令                   │
│ □ 任务内容已在SOUL.md白名单范围内              │
└──────────────────────────────────────────────┘
```

## 二、任务模板库（按助手分类）

### 2.1 千问任务模板（代码/数学/逻辑）

#### 【T-QW-01】策略代码审查
```
【角色】你是量化交易策略代码审查助手

【输入】用户提供策略代码片段或文件路径

【审查要点】
1. 逻辑正确性：条件判断/循环/异常处理
2. 数据泄露风险：全局变量/闭包/缓存
3. 性能瓶颈：循环嵌套/重复计算/内存占用
4. Token节省：可复用代码/缓存策略/批量处理
5. 安全性：输入校验/边界检查/权限控制

【输出格式】
## 代码审查报告
### 问题列表
- [P0/严重] ...
- [P1/警告] ...
- [P2/建议] ...
### Token节省建议
### 总体评价
```
**触发关键词**：`代码审查` `review` `检查代码` `优化建议` `代码问题`
**预计节省Token**：90%+（外脑截图分析 vs QClaw全量解析）
**超时设置**：30秒，超时自动降级QClaw处理

## 三、多助手协作模式

### 3.1 投票仲裁模式（Vote Arbitration）

**触发场景**：
- 策略调整方向不确定
- 新标的重大决策
- 市场判断分歧

**协作流程**：
```
任务输入
    │
    ├── 豆包搜索 → 情报视角
    │
    ├── 千问分析 → 策略视角
    │
    └── 元宝研判 → 研报视角
            │
            ▼
    QClaw综合三方意见 → 最终决策
```

**示例Prompt（Vote模式）**：
```
请从以下三个角度分析这个交易决策：

【豆包视角】分析该标的的市场情绪和近期动态
【千问视角】分析该策略信号的逻辑合理性和风险
【元宝视角】综合研报信息判断基本面支撑

最终请给出你的综合判断和置信度。
```

---

### 3.2 串联模式（Chain）

**触发场景**：
- 复杂任务需要多步骤处理
- 上一输出作为下一输入

**协作流程**：
```
豆包搜索情报 → 元宝整理研判 → 千问策略验证 → QClaw决策
```

**示例Prompt（Chain模式）**：
```
Step 1（豆包）：搜索该行业最新动态
Step 2（元宝）：基于搜索结果，判断整体情绪
Step 3（千问）：基于情绪分析，评估交易信号有效性
Step 4（QClaw）：综合判断，决定是否执行
```

---

### 3.3 并行模式（Parallel）

**触发场景**：
- 多个独立子任务
- 需要快速获取多维度信息

**协作流程**：
```
    ├── 助手A → 子任务1 ──┐
    ├── 助手B → 子任务2 ──┼→ QClaw整合
    └── 助手C → 子任务3 ──┘
```

**示例Prompt（Parallel模式）**：
```
并行执行以下三个独立任务：
1. 豆包：搜索该公司的最新公告
2. 千问：分析当前持仓风险
3. 元宝：整理相关研报要点

完成后请汇总三个结果。
```

## 五、异常处理与降级

### 5.1 外脑失效降级规则

| 失效情况 | 降级策略 | Token节流 |
|---------|---------|---------|
| 单助手超时 | 切换其他助手重试1次 | 启用 |
| 单助手报错 | 尝试其他助手，仍失败则QClaw处理 | 启用 |
| 全部助手不可用 | QClaw自身处理，记录日志，发送告警 | 强制启用 |
| 响应质量低 | 记录质量评分，累积3次触发路由调整 | 启用 |

### 5.2 决策隔离规则（SOUL.md强制要求）

```
外脑回复 → QClaw二次校验 → 人工确认（重大决策）→ 执行
         ↑
    必须经过此步骤！
```

**无需人工确认**（低风险）：
- 信息检索/情报收集
- 代码审查/建议
- 性能分析/总结

**需要人工确认**（高风险）：
- 新策略部署
- 大额调仓
- 风控参数变更
- 涉及资金变动的任何建议

## 六、Token效率追踪

### 6.1 效率计算公式

```
效率 = 产出价值 / Token消耗

产出价值评估：
- 代码审查：避免潜在bug → 高价值
- 市场情报：快速决策支持 → 高价值
- 研报摘要：节省阅读时间 → 中价值
- 策略建议：决策参考 → 中价值
```

### 6.2 效率基线（预期值）

| 任务类型 | QClaw直接处理 | 外脑桥接 | 节省比例 |
|---------|--------------|---------|---------|
| 代码审查 | 800K | 50K | 94% |
| 市场情报 | 500K | 30K | 94% |
| 研报摘要 | 1.2M | 100K | 92% |
| 策略分析 | 600K | 80K | 87% |

### 6.3 效率监控指标

- **日均外脑调用次数**：目标≥10次/日（非交易时段）
- **外脑Token占比**：交易时段≥40%，非交易时段≥70%
- **调用成功率**：目标≥99%
- **响应质量评分**：1-5分，目标≥4分
