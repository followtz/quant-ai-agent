## 成本控制手段（已落实）

| 措施 | 节省效果 | 状态 |
|------|---------|------|
| 巡检零LLM化（health_check.py代替Heartbeat） | ~96%巡检Token | ✅ 已执行 |

| 缓存复用（session上下文保持） | ~50% 重复查询 | ✅ OpenClaw原生 |
| 纯代码执行（market_briefing/auto_backtest/futu_data_dl） | 零Token执行 | ✅ 已执行 |
| 子智能体隔离（短session，不积累上下文） | 避免上下文膨胀 | ✅ 架构设计 |

## 审计方式

```
每日: session_status 精确查询 → 记录到 data/history/token_usage.csv
每周: 汇总Token消耗 → 对比分配计划
每月: 评估投入产出 → 调整下月分配
```
