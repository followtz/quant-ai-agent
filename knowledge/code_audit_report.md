## 二、连连数字V4 实盘引擎

**文件**: `v4_live_engine.py` (570行)

### 现状
OOP设计的三重确认策略引擎（V3+均值回归+ML），约570行。

### 发现的问题

| 问题 | 严重度 | 说明 |
|------|--------|------|
| **ML模型数据过少** | 🔴 阻塞 | 60天数据训练RandomForest+6特征≈严重过拟合，预测≈随机 |
| **港股判断bug** | 🔴 阻塞 | `is_hk_pre = (9 <= hk_hour < 9)` 永远为False，死代码 |
| **while True无超时** | 🟡 中 | 无限循环可能hang，需加最大运行时间保护 |
| **CSV回退路径** | 🟡 中 | `data/02598_daily.csv` 相对路径，依赖启动目录 |
| **模拟/实盘混合** | 🟢 低 | FUTU_AVAILABLE标志控制，但容易被误用 |
| **60秒轮询** | 🟢 低 | 港股每分钟检查一次，非交易时段浪费API额度 |

### 优化建议

```
🔴 紧急:
  1. 移除ML信号（RandomForest在60条数据上≈随机猜测）
  2. 修复 is_hk_pre bug（虽然不影响，但代码有误导）
  3. 改用auto_backtest.py的backtrader做回测验证

🟡 建议:
  4. 合并V3+均值回归为双信号（比三重确认更可靠）
  5. 非交易时段不轮询（用cron触发替代while True）
  6. 路径改用Path(__file__).resolve().parent

🟢 可选:
  7. 日志改用标准logging（不用重复造轮子）
  8. 加入OpenClaw cron替代轮询
```

### ML信号的真相
用60天数据训练RandomForest，6个特征（包含重复信息如macd/macd_signal），预测准确率≈50%（扔硬币）。**建议移除**，回归到V3+均值回归双信号。不仅更可靠，而且省掉了sklearn依赖和ML训练Token消耗。

### 核心原则
```
更少的信号 + 更好的执行 > 更多的信号 + 混乱的决策
```
