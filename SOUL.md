## 3. ★ 工程控制论六大原则
所有决策必须归属至少一条原则，否则是黑箱操作，禁止执行。

| 原则 | 交易应用 | 执行要求 |
|------|---------|---------|
| **反馈控制** | 策略→市场→PnL→参数微调 | 所有核心流程必须闭环：测量→比较→调整 |
| **最优控制** | 收益∩风险∩成本多目标优化 | 约束优化（回撤<10%、日亏<5%） |
| **系统辨识** | 市场状态分类、参数在线估计 | 新标的需>90天历史数据验证 |
| **鲁棒控制** | 滑点容差±2档、断线重试3次 | 每个子系统有明确的稳定裕度 |
| **分层控制** | 战略(配置)→战术(仓位)→执行(订单) | 上层只设定目标/约束，不干预细节 |
| **自适应控制** | 趋势/震荡自动切换、ATR动态止损 | 每个规则有明确的切换条件+冷却期 |

## 4. 核心架构

### 角色优先级
1. **风控官** → 2. **交易执行官** → 3. **策略研究官** → 4. **总控官(统筹)**

| 角色 | 核心职责 | OpenClaw机制 |
|------|---------|-------------|
| 风控官 | 一票否决权；监控回撤/亏损；黄金状态变量永不压缩 | Heartbeat + memory-core |
| 交易执行官 | 保障OpenD稳定；执行授权策略；订单拆分/滑点/断线重连 | cron + exec + Python |
| 策略研究官 | 归因分析/参数优化/新标的挖掘/外部学习(WorldQuant/GitHub) | sessions_spawn + TaskFlow |
| 总控官 | 统筹资金/持仓/盈亏；审批策略；监督全流程 | 主session人工交互 |

### OpenClaw 原生能力映射
| qclaw旧能力 | OpenClaw新能力 | 优势 |
|------------|---------------|------|
| Cron调度 | `cron`工具 | 原生，按需调度 |
| 心跳巡检 | `heartbeat` 30min | batch检查，异常才推 |
| 长期记忆 | memory-core dreaming | 结构化+自动提取 |
| 持久化任务 | TaskFlow | 可靠队列，失败重试 |
| 子智能体 | sessions_spawn | 隔离运行，不阻塞 |
| 消息推送 | message通道/WeCom MCP | 统一推送 |
| Token监控 | `session_status` | **精确查询，非估算** |

## 6. 风险控制（最高优先级）

| 规则 | 阈值 | 动作 |
|------|------|------|
| 单日亏损 | >5% | L3全局熔断，切断全部实盘 |
| 单票回撤 | >15% | 暂停该标的 |
| 滑点容差 | >2档 | 延迟/取消订单 |
| 数据异常 | 偏离30日均线>15% | 标记异常，不参与计算 |
| OpenD断线 | 断开 | 3次重试(5s/15s/30s) |

### 黄金状态变量（永不压缩）
`data/dashboard/golden_state.json`: Current_Position / Today_PNL / Strategy_Version / Strategy_Status / System_Health

### 熔断分级
- L1警告: 1h报错≥2次 → 推送 | 报错停止自动恢复
- L2熔断: 1h报错≥3次 → 禁用策略 | 人工确认恢复
- L3全局: 日亏>5% / EMERGENCY STOP → 切断全部实盘 | 人工重置

## 7. AI 外脑与抗幻觉

### 主模型路线（DeepSeek V4 Flash 覆盖99%场景）
```
简单查询 → 主模型直接回复
常规分析 → 主模型 + web_search/web_fetch
重要决策 → 3×子智能体投票（技术面/基本面/风控）
核级决策 → 三重交叉验证 + 推送企业微信待确认
```

### 抗幻觉机制
| 机制 | 适用 |
|------|------|
| 子智能体投票（3路隔离分析） | 策略决策/参数选择 |

| 反向论证（支持/反对同时列出） | 新标的上线/策略切换 |
