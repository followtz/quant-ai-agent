# AI外脑路由SOP - 典型任务自动路由手册
> 符合SOUL.md第六章AI外脑协同规范，v1.1（2026-04-23）
> 
> **核心原则**：「先风控、再执行、后进化」「外脑建议需经QClaw二次校验」「严禁直接下单」
> **重要调整**：基于2026-04-23实测结果，千问升主力、豆包降备用、元宝并列主力

---

## 一、路由决策树（优先级从高到低）

### 第一层：任务类型分类

```
用户请求/任务触发
       │
       ├── 包含以下关键词？──→ 直接执行/拒绝
       │   「下单」「买入」「卖出」「平仓」「开仓」
       │   「API Key」「密码」「密钥」「账户」
       │   「熔断」「切断」「STOP ALL」「EMERGENCY」
       │   → QClaw自身处理（核心决策/安全敏感/风控类）
       │
       ├── 以下关键词任一触发 ──→ 千问路由（首选）
       │   「代码」「Python」「回测」「策略逻辑」「算法」
       │   「review」「审查」「bug」「报错」「修复」
       │   「因子计算」「数学」「统计」「回归」「优化」
       │   「参数」「改进」「归因」「建议」「方案」
       │
       ├── 以下关键词任一触发 ──→ 元宝路由（首选）
       │   「研报」「报告」「PDF」「文档」「总结」
       │   「摘要」「长文」「解读」「分析」「综合」
       │   「年报」「招股书」「公告全文」「多文档」
       │   「新闻」「市场」「行情」「最新」「今日」「搜索」
       │   「舆情」「行业」「板块」「情绪」
       │
       ├── 以下关键词任一触发 ──→ 豆包路由（备用，限流时不可用）
       │   「创意」「脑洞」「发散」「联想」「头脑风暴」
       │   「竞品」「对比」「差异化」「独特」
       │
       └── 复合任务 ──→ 多助手投票/并行
           「这个策略怎么样」→ 千问代码review + 元宝长文
           「帮我分析这几个标的风险」→ 千问计算 + 元宝研判
           「最近的交易机会」→ 元宝情报 + 千问策略
```

### 第二层：安全边界校验（所有外脑任务必须通过）

```
┌──────────────────────────────────────────────┐
│ 安全边界检查（不符合任一则拒绝外脑处理）        │
├──────────────────────────────────────────────┤
│ □ 不含账户信息（持仓金额/成本/盈亏细节）        │
│ □ 不含个人信息（姓名/手机/邮箱/地址）          │
│ □ 不含API密钥/Token/凭证                      │
│ □ 不含交易指令（下单价/数量/买卖方向）          │
│ □ 输出不含直接执行的操作命令                   │
│ □ 任务内容已在SOUL.md白名单范围内              │
└──────────────────────────────────────────────┘
```

## 二、助手能力矩阵（v1.1 实测更新版）

| 助手 | 核心能力 | 分配任务 | 触发关键词 | 安全边界 | Token优化策略 | 可用性评级 |
|------|----------|----------|------------|----------|---------------|-----------|
| 千问 | 代码/数学/逻辑/改进 | 代码审查、策略Debug、因子计算、风控审计、参数优化、策略改进 | 代码、Debug、计算、Python、风控、review、报错、优化、改进 | 禁止传输敏感信息,生成代码需沙箱校验 | 优先复用历史审查成果,同类任务直接调用缓存结果 | ⭐⭐⭐⭐⭐ 主力 |
| 元宝 | 长文本/总结/搜索/情报 | 研报解读、日志摘要、多文档分析、市场情报、新闻搜索、情绪分析 | 研报、总结、摘要、分析、PDF、公告、年报、新闻、市场、情绪 | 仅处理脱敏数据,输出需精简核心信息 | 摘要任务设定固定Token上限,长文本分段处理 | ⭐⭐⭐⭐ 主力 |
| 豆包 | 创意/发散/竞品分析 | 头脑风暴、创意方案、竞品对比（限流时不可用） | 创意、脑洞、发散、联想、竞品、对比 | 禁止敏感信息 | 批量处理同类搜索任务,结果缓存复用 | ⭐⭐⭐ 备用 |

**【v1.1 重要调整】基于2026-04-23实测**：
- **千问升级为主力**：DOM稳定(e166/e167)、复制按钮可用(e164)、代码能力强、无服务限流
- **元宝并列主力**：发送稳定(press Enter)、适合研报/摘要/情报，无复制按钮需截图提取
- **豆包降级为备用**：频繁"服务太火爆"限流，仅作为千问/元宝不可用时 fallback

## 三、路由优先级速查表（v1.1 更新）

| 任务场景 | 首选助手 | 备选 | 需要投票？ |
|---------|---------|------|----------|
| 代码审查 | 千问 | - | 否 |
| 回测报错 | 千问 | - | 否 |
| 因子计算 | 千问 | - | 否 |
| 参数优化 | 千问 | 元宝 | 视复杂度 |
| 策略改进 | 千问 | 元宝 | 视复杂度 |
| 新闻搜索 | 元宝 | 豆包 | 否 |
| 市场情绪 | 元宝 | 豆包 | 否 |
| 研报摘要 | 元宝 | - | 否 |
| 公告解读 | 元宝 | 千问 | 否 |
| 多文档分析 | 元宝 | 千问 | 是 |
| 新标的筛选 | 元宝 | 千问 | 是 |
| 重大决策 | 全助手 | - | 是（必须） |
| 风控审计 | 千问 | 元宝 | 否 |
| 策略归因 | 千问 | 元宝 | 否 |
| 日报生成 | 元宝 | 千问 | 否 |
| 日志分析 | 千问 | 元宝 | 否 |
| 头脑风暴 | 豆包 | 千问 | 否 |

**【v1.1 调整说明】**：
- 移除豆包作为新闻/市场情绪首选（限流风险高）
- 元宝承担原豆包的新闻/市场情绪类任务
- 千问承担更多策略改进/归因类任务
- 豆包仅保留创意/发散类任务
