## 一、版本命名规范

### 1.1 语义化版本号

格式：`Major.Minor.Patch[-Label]`

| 字段 | 含义 | 递增规则 |
|------|------|---------|
| Major | 主版本 | 策略逻辑根本性重构（如交易信号生成机制改变） |
| Minor | 次版本 | 参数优化、新增过滤条件、风控规则调整 |
| Patch | 补丁版本 | Bug修复、日志格式调整、非逻辑性改动 |
| Label | 预发布标签 | alpha / beta / rc / hotfix |

**标签定义**：

| 标签 | 含义 | 允许实盘 | 说明 |
|------|------|---------|------|
| alpha | 内部测试 | ❌ 禁止 | 代码逻辑验证阶段 |
| beta | 回测验证 | ❌ 禁止 | 历史回测通过，影子模式验证中 |
| rc | 候选发布 | ❌ 禁止 | 影子模式通过，待总控审批 |
| （无标签） | 正式发布 | ✅ 允许 | 总控审批通过，可接入实盘 |
| hotfix | 紧急修复 | ⚠️ 限速 | 实盘紧急修复，限制仓位运行 |

### 1.2 命名示例

```
prev_close_v2.0.0       # BTDR PrevClose V2 第一个正式版
prev_close_v2.1.0-beta  # V2.1 回测验证中
prev_close_v2.1.0       # V2.1 正式发布
prev_close_v2.1.1-hotfix # V2.1 紧急修复
prev_close_v3.0.0-alpha # V3 逻辑重构，内部测试

lianlian_v4.0.0         # 连连数字V4 正式版
lianlian_v4.1.0-rc      # V4.1 候选发布
```

### 1.3 文件命名规范

| 文件类型 | 命名格式 | 示例 |
|---------|---------|------|
| 策略引擎 | `{strategy}_{version}_engine.py` | `prev_close_v2.1.0_engine.py` |
| 配置文件 | `{strategy}_{version}_config.json` | `prev_close_v2.1.0_config.json` |
| 回测数据 | `{strategy}_{version}_backtest_{date}.json` | `prev_close_v2.1.0_backtest_20260420.json` |
| 回测报告 | `{strategy}_{version}_report_{date}.md` | `prev_close_v2.1.0_report_20260420.md` |
| 变更日志 | `CHANGELOG.md` | 统一文件，追加记录 |

## 二、版本归档标准

### 2.1 归档目录结构模板

```
01_策略库/{策略名}/模型迭代版本归档/
├── v{Major}.{Minor}.{Patch}[-Label]/
│   ├── engine/                          # 策略引擎代码
│   │   └── {strategy}_{version}_engine.py
│   ├── config/                          # 配置文件
│   │   └── {strategy}_{version}_config.json
│   ├── backtest/                        # 回测数据与报告
│   │   ├── {strategy}_{version}_backtest_{date}.json
│   │   └── {strategy}_{version}_report_{date}.md
│   ├── CHANGELOG.md                     # 版本变更记录
│   └── README.md                        # 版本说明
```

### 2.2 归档必含文件清单

| 文件 | 必须性 | 说明 |
|------|--------|------|
| 策略引擎代码 | ✅ 必须 | 可运行的完整策略代码 |
| 配置文件 | ✅ 必须 | 包含所有参数的配置JSON |
| 回测报告 | ✅ 必须 | 至少一份完整回测报告（含收益曲线、最大回撤、夏普比率） |
| CHANGELOG.md | ✅ 必须 | 详细记录本次版本变更内容 |
| README.md | ✅ 必须 | 版本概述、关键变更、已知问题 |

### 2.3 不可变原则

> **已归档版本禁止任何修改！**

- ✅ 允许：读取、引用、复制为新版本基础
- ❌ 禁止：修改已归档的代码、配置、回测数据
- ❌ 禁止：覆盖已归档文件
- ❌ 禁止：删除已归档版本目录

**如果已归档版本发现问题**：
1. 创建新的hotfix版本（如 v2.1.1-hotfix）
2. 在新版本CHANGELOG中说明修复的归档版本问题
3. 保留原归档版本不动

## 三、版本迭代与回滚流程

### 3.1 新版本发布流程

```
1. 开发阶段
   ├── 从最新正式版复制代码
   ├── 修改版本号和配置
   └── 标记为 alpha
   
2. 回测验证
   ├── 运行历史回测（至少1年数据）
   ├── 回测指标达标（胜率>55%，夏普>1.0，最大回撤<15%）
   └── 标记为 beta
   
3. 影子模式
   ├── 在实盘环境运行Paper Trading ≥ 2周
   ├── 记录模拟信号与实盘差异
   ├── 偏差分析报告
   └── 标记为 rc
   
4. 总控审批
   ├── 提交版本发布申请
   ├── 附回测报告 + 影子模式报告
   ├── 总控审批通过
   └── 移除标签，正式发布
   
5. 实盘上线
   ├── 小仓位运行1周（1/3标准仓位）
   ├── 无异常 → 标准仓位运行
   └── 归档至模型迭代版本归档目录
```

### 3.2 版本回滚

**触发条件**：

| 条件 | 严重程度 | 动作 |
|------|---------|------|
| 实盘连续3天亏损 | 中 | 评估是否回滚 |
| 单日亏损 > 3% | 高 | 立即评估回滚 |
| 策略逻辑异常（非预期交易） | 紧急 | 立即回滚 |
| 单日亏损 > 5% | 紧急 | 立即回滚 + 风控熔断 |

**回滚操作步骤**：

1. 立即停止当前版本实盘交易
2. 从归档目录复制上一正式版引擎和配置
3. 验证回滚版本代码完整性（对比归档MD5）
4. 以1/3仓位重启回滚版本
5. 观察1个交易日无异常 → 恢复标准仓位
6. 记录回滚原因至CHANGELOG
7. 失败版本标记问题，修复后重新走发布流程

### 3.3 版本废弃标准

满足以下任一条件的版本可标记为废弃：

- 被更新的正式版完全替代超过3个月
- 存在已知严重Bug且已有修复版
- 策略逻辑已被证明无效（回测/实盘均表现不佳）

废弃操作：
- 在目录名添加 `_deprecated` 后缀
- 在README.md标注废弃原因和替代版本
- 不删除，保留供历史参考

## 四、变更日志模板（CHANGELOG.md）

```markdown
# Changelog

## [2.1.0] - 2026-04-20

### Added
- 新增成交量过滤条件：排除成交量低于20日均值50%的信号
- 新增开盘跳空阈值参数：gap_threshold

### Changed  
- 调整持仓时间上限从120分钟至90分钟
- 优化止损逻辑：从固定比例改为ATR动态止损

### Fixed
- 修复港股收盘前5分钟未强制平仓的问题
- 修复日志时间戳时区错误（UTC→HKT）

### Removed
- 移除废弃的V1信号生成逻辑

- 胜率：58.3% (V2.0: 55.1%)
```
